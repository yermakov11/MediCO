Тема веб додатку - «Облік замовлень інтернет-магазину»

Створити веб додаток з реєстрацією/авторизацією/аутентифікацією (2 актори: менеджер, пакувальник)
Менеджер:
- можливість додавати, редагувати, видаляти замовлення та продукти
Пакувальник:
- можливість переглядати списки продуктів та замовлень
- можливість змінювати статус замовлення

Вимоги:
- Описати та продемонструвати: діаграму варіантів використання, діаграму класів додатку

План разробки 

Крок 1: Підготовка середовища
1.1. Встановити Node.js та npm на вашому комп'ютері.+
1.2. Створити новий каталог для проекту та ініціалізувати проект за допомогою npm init.+

Крок 2: Налаштування серверної частини (Express.js)
2.1. Встановити Express.js, як залежність проекту за допомогою npm install express.
2.2. Створити серверну структуру проекту, включаючи файли для маршрутів, моделей і контролерів.
2.3. Створити базу даних (наприклад, використовуючи SQLite або MongoDB) та підключитися до неї.
2.4. Реалізувати авторизацію/аутентифікацію для менеджера та пакувальника, можливо, 
    за допомогою Passport.js або іншої аутентифікаційної бібліотеки.
2.5. Створити маршрути для додавання, редагування, видалення замовлень та продуктів для менеджера.
2.6. Створити маршрути для перегляду списків продуктів та замовлень для пакувальника, 
а також для зміни статусу замовлення.

Крок 3: Розробка клієнтської частини (React)
3.1. Створити React-проект за допомогою Create React App або іншого інструменту.
3.2. Розробити компоненти для реєстрації, авторизації і аутентифікації користувачів.
3.3. Розробити компоненти для менеджера та пакувальника, які включають форми для додавання, редагування, видалення замовлень та продуктів.
3.4. Розробити компоненти для пакувальника для перегляду списків продуктів та замовлень, а також для зміни статусу замовлення.
3.5. Забезпечити маршрутизацію між сторінками та обробку подій для взаємодії з серверною частиною.

Крок 4: З'єднання серверної та клієнтської частин
4.1. Спільно встановити CORS (Cross-Origin Resource Sharing) для дозволу запитів від клієнта до сервера.
4.2. Зробити запити до серверної частини для обробки даних і відображення їх на клієнтському інтерфейсі.
